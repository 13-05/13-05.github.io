Making Pacman Easy

In this post, I discuss quality-of-life (QOL) things related to pacman, like aliases and functions for certain things.

For new users on Arch based distros, pacman and all its arguments might seem a little confusing. Or, veteran users might feel like it's too much to type. Well, I feel both of those ways (although I'm not a veteran), so here are some QOL things I have that makes everything (or, most things) easy.

Here are the x-super-installer and x-super-remover functions, basically interactive pacman searches to install or uninstall packages (credit to this post, as this is just a port of that function to Arch!):

xsi () {
	pacman -Slq | fzf -m --preview-window="right:66%:wrap" --preview "pacman -Si {1}" | xargs -ro sudo pacman -S
}
xsr () {
	pacman -Qq | fzf -m --preview-window="right:66%:wrap" --preview "pacman -Q --info {1}" | xargs -ro sudo pacman -R
}

Here are some aliases I find useful (heavily inspired by this blog post):

alias xi="sudo pacman -S"
alias xr="sudo pacman -Rns"
alias xq="sudo pacman -Si"

Finally, my must-have package on any Arch-based system is yay. Instead of running sudo pacman -Syu every time you need to update stuff, just type yay! Also, it syncs up the AUR, as well as working as an AUR package manager for directly installing things. Overall, yay's just great.
